How To Use The API:

  • To Add New User, please head to: Homepage/

  • To Write new Message, please head to: WriteMessage/

  • To Read a Message, please head to: ReadMessage/

  • To Get Messages for a Specific User (All or Unread), please head to: GetMessages/

  • To Delete a Message, please head to: DeleteMessage/

  • To see this Index Page again later, please return to: Index/

Some Notes about the API:
  • Every Message in the data base of the system do not have an indication who is the owner of the message, but the sender and the receiver. It can cause a problem while a User tries to delete a message (as a sender or as a receiver) - bacause it will automaticly delete the message from the system, no matter that there is another User in the system with the same message (as a receiver or as a sender).
  • The problem above could be solved easily with more time, and another Model.Field - that will give indication who is the OWNER of the message, and then create a DUPLICATE of that message in the system, while in the first message - the owner is the sender, and in the other message - the owner is the receiver. This solution will take some time that I do not have right now...
  • Another thing, is that the system CAN NOT handle a situation that a User send or receive a message with the same subject... It will cause a CONFLICT, so please - DON'T TRY IT!
